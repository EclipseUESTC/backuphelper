# 需求规格说明 (C++版本)

## 1. 基本功能需求
- [x] 备份指定目录树到目标路径
- [x] 从备份路径还原目录树到指定位置

## 2. 扩展功能需求

### 2.1 文件类型支持
- [x] 支持备份和还原以下特殊文件：
  - 符号链接（soft link）
  - 硬链接（hard link）
  - 管道文件（FIFO）
  - 设备文件（字符/块设备）
  - 套接字文件

### 2.2 元数据支持
- [x] 保留以下文件属性：
  - 文件权限
  - 创建/修改时间
  - 文件属主和属组

### 2.3 自定义备份
- [x] 支持按以下条件筛选备份文件：
  - 路径匹配
  - 文件类型
  - 文件名（支持通配符）
  - 文件大小范围
  - 时间范围（创建/修改时间）

### 2.4 打包解包
- [x] 支持将多个文件打包为单个备份文件
- [x] 支持从打包文件中解包还原

### 2.5 压缩解压
- [x] 支持 Haff 和 LZ77 压缩算法
- [x] 可选择是否启用压缩

### 2.6 加密解密
- [x] 支持 AES / DES 加密算法
- [x] 用户可设置加密密钥

### 2.7 图形界面
- [x] 基于 Qt 的图形化操作界面
- [x] 支持路径选择、条件设置、算法选择等
- [x] 显示备份/还原进度和结果

## 3. 技术栈与依赖

### 3.1 核心依赖
- **编译器**: GCC/Clang/MSVC (C++17)
- **构建系统**: CMake 3.16+
- **图形界面**: Qt 5.15+
- **文件系统**: std::filesystem (C++17)

### 3.2 可选依赖
- **压缩库**: zlib (备用压缩实现)
- **加密库**: OpenSSL (加密算法支持)
- **测试框架**: Google Test

### 3.3 平台支持
- Windows (MSVC/MinGW)

## 4. 非功能性需求
- 界面友好、操作简便
- 备份/还原过程可中断
- 日志记录操作过程
- 内存使用优化，支持大文件处理

## 5. 开发规范
- 代码风格：遵循 Google C++ Style Guide
- 版本控制：Git + 语义化版本
- 文档：Doxygen 生成 API 文档
- 测试：单元测试覆盖率 > 80%
