# 补充知识
## Unix 文件系统
> Unix系统中存在的文件类型
>   - 普通文件（-）存储数据的容器
>   - 目录文件（d）文件夹
>   - 字符设备文件（c）键盘、终端等
>   - 块设备文件（b）硬盘、U盘等
>   - 管道文件（p）进程间通信通道
>   - 符号链接文件（l）文件的“快捷方式”
>   - 套接字文件（s）网络通信端点

### 普通文件 
> 就像Windows中.txt .exe .docx一样，是最常见的文件
- 用 ls -l 可以查看普通文件的权限、属主、时间、尺寸等信息。
- 普通文件用 - 开头

### 目录
> 就像Windows中的文件夹一样，是一种特殊的文件
- 用 ls -l 可以查看目录的权限、属主、时间、尺寸等信息。
- 目录用 d 开头

### 字符设备
> 就像Windows中的串口、鼠标、键盘等一样，但是Windows通常隐藏这些细节；Linux把设备文件当做文件暴露
- 用 ls -l 可以查看字符设备的权限、属主、时间、尺寸等信息。
- 字符设备用 c 开头

### 块设备
> 就像Windows中的硬盘、U盘等一样“按块读写”的存储设备
- 用 ls -l 可以查看块设备的权限、属主、时间、尺寸等信息。
- 块设备用 b 开头

### 套接字
> Windows也有socket，但通常不表现为文件；Linux本地进程通信可用"Unix 域套接字"，表现为文件
- 用 ls -l 可以查看套接字的权限、属主、时间、尺寸等信息。
- 套接字用 s 开头

### 管道
> 就像Windows中的“命名管道”，用于进程间通信
- 用 ls -l 可以查看管道的权限、属主、时间、尺寸等信息。
- 管道用 p 开头

### 链接文件：硬链接
- 类似给文件“取别名”，指向同一个inode（即同一个文件数据）
- 与源文件完全平等，无主次之分
- 可以通过 ls -i 查看文件的inode号
  
### 典型Unix 磁盘布局
> 硬盘被氛分为多个“划分”（Partition）和一个Boot Sector。
- 划分(Partition)：将磁盘划分为多个区域，每个区域有一个文件系统和一个Boot Sector。
- Boot Sector：引导扇区，用于引导操作系统加载。
- 文件系统(ext2)
- ext2文件系统被划分为多个块组(Block Group)，每个块组包含多个块(Block)。
  - 如果把文件系统比作行政市，那么块组就相当于一个行政区。
- 每个块组有一个超级块(Super Block)，用于存储文件系统的元数据。
  - 超级块类似区政府的总体规划，记录了文件系统的整体信息。
    - 如总人口（inode 总数）、总土地面积（block 总数）、行政区数量、每块地多大（block 大小）、文件系统状态（是否干净卸载）等
- 每个块组有一个块组描述符表(Block Group Descriptor)，用于存储块组的元数据。
  - 块组描述符类似区的行政区划图。
    - 标明每个行政区（块组）的详细信息：该区有多少空地（块位图位置）、档案室在哪（inode位图位置）、身份证号池状态（inode表起始块号）等
- 每个块组有一个块位图(Block Bitmap)，用1位表示一个数据块是否被占用。
  - 分配数据时，优先在同一个块组内分配
- 每个块组有一个inode位图(Inode Bitmap)，用于记录该块组哪些inode已被使用。
- 每个块组有一个inode表(Inode Table)，用于存储文件的元数据。
  - inode表类似居民档案室，存储了文件类型、权限、所有者、时间戳、数据块指针、链接计数等等
- 每个块组有一个数据块区域(Data Block Region)，用于存储文件的数据。
